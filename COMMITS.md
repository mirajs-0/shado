# Shado — Commit Convention

All commits follow the Conventional Commits standard.

## Format

type: short description

## Types

| Type     | When to use                        |
| -------- | ---------------------------------- |
| feat     | new feature or functionality       |
| fix      | bug or error fix                   |
| docs     | documentation changes              |
| style    | formatting, no logic change        |
| refactor | rewrite without changing behaviour |
| test     | adding or updating tests           |
| chore    | setup, config, tooling             |
| learn    | lesson code and practice files     |

## Rules

- always lowercase
- no full stop at the end
- present tense
- under 72 characters
- never vague

## Examples

feat: add behavioral data collector script
fix: correct app sequence matching logic
learn: lesson 3 dictionaries nested profiles
chore: initialise project folder structure
docs: update README with tech stack
